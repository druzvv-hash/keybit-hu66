# Security Policy

## Supported version

Only the latest public alpha is actively maintained.

## Reporting a software vulnerability

Use GitHub's private security advisory feature for vulnerabilities that could expose local files, bypass intended browser isolation, or silently transmit data. Do not open a public issue with exploit details before a fix is available.

## Physical-security and misuse reports

KeyBit is intended for owners and authorized automotive/locksmith professionals. The project does not include a key-code database and should not be used to facilitate unauthorized access.

Do not attach identifiable customer keys, VINs, addresses, or proprietary database exports to a public report.

## Result integrity

A wrong but high-confidence bitting candidate is treated as a safety defect. Reports should include the software version, device/browser, photo conditions, alignment mode, displayed confidence, and independently verified expected result. Use synthetic or explicitly authorized evidence only.
