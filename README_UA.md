# KeyBit HU66

**Експериментальний повністю локальний декодер внутрішньої доріжки HU66 для браузера телефона або комп’ютера.**

KeyBit вирівнює фото леза, нормалізує робочу область HU66, витягує видиму траєкторію доріжки, оцінює вісім позицій бітингу та застосовує перевірку HU66 MACS. Поточний публічний реліз — **дослідна alpha-версія**, а не остаточне джерело даних для різання.

## Навіщо цей проєкт

Комерційні станки зручні, але сам процес комп’ютерного розпізнавання ключа є цікавою інженерною задачею. KeyBit досліджує прозорий офлайн-підхід, який можна перевіряти, тестувати й покращувати відкрито.

## Що вже працює

- Повністю локальна обробка в JavaScript; застосунок сам не завантажує фото на сервер.
- Український та англійський інтерфейс.
- Автоматичний і ручний поворот леза.
- Ручна рамка за двома точками: **кінчик** і **плече**.
- Трафарет HU66 з поточною геометрією 8,18 × 28 мм та позиціями 3–24 мм.
- Виділення траєкторії внутрішньої доріжки та оцінка впевненості кожної позиції.
- Перевірка HU66 MACS із окремим показом скоригованого кандидата.
- Збереження debug-зображення та JSON.
- Автономний однофайловий build без сервера й встановлення залежностей.

## Як запустити

### Через GitHub Pages

Після ввімкнення GitHub Pages для кореня репозиторію відкрий Pages-посилання на телефоні або комп’ютері.

### Локально

Завантаж `index.html` або версію з `dist/` і відкрий у браузері. Для стабільнішої роботи браузерного origin можна запустити простий локальний сервер:

```bash
python -m http.server 8000
```

Після цього відкрий `http://localhost:8000`.

## Робочий процес

1. Зроби чітке фото власного або авторизованого ключа HU66.
2. Натисни **Автовирівнювання** та доведи лезо по вертикалі ручним поворотом.
3. Якщо автоматична геометрія ненадійна, ввімкни ручну рамку.
4. Постав **КІНЧИК** у центр торця, а **ПЛЕЧЕ** — на початок 28-мм робочої частини.
5. Запусти декодування й перевір траєкторію, впевненість позицій, сирий результат та MACS.
6. Перед різанням обов’язково звір бітинг незалежним авторизованим методом.

## Обмеження

- Це alpha з малим датасетом, а не комерційна валідація.
- Освітлення, відблиски, зношення, перспектива, варіанти леза й обробка камери можуть зміщувати видимий край.
- Правдоподібний код усе одно може бути неправильним.
- MACS — лише фільтр логічності, а не доказ правильності.
- Поточна публічна версія підтримує лише HU66 internal-track.
- Бази заводських або комерційних кодів у репозиторії немає.

## Приватність

У застосунку немає телеметрії, аналітики, endpoint для завантаження чи віддаленої моделі. Фото обробляється в браузері. При розміщенні на GitHub Pages сама платформа хостингу може вести звичайні журнали доступу, але KeyBit не отримує вибране фото ключа на сервері.

Детальніше: [docs/PRIVACY_UA.md](docs/PRIVACY_UA.md).

## Лише авторизоване використання

Використовуй KeyBit тільки для ключів, замків і автомобілів, що належать тобі або які ти маєш право обслуговувати. Не публікуй клієнтські фото ключів, VIN, реальні пари «фото ↔ бітинг», EXIF-геолокацію або скопійовані комерційні бази.

## Перевірка

Публічний репозиторій навмисно не містить клієнтських фотографій і приватного regression-датасету. Внутрішня alpha перевірена на невеликому наборі авторизованих пар «ключ ↔ станок», але це не є виробничою гарантією.

Перевірка цілісності build:

```bash
npm test
```

Вона перевіряє синтаксис JavaScript, наявність двомовного UI, компонентів вирівнювання/MACS, відсутність зовнішніх runtime-URL та приватних regression-ідентифікаторів.
